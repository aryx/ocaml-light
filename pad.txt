* History

** first try: caml special light 1.16, May 1996? 19 years ago ...

I originally started my fork from caml special light 1.16 just before
the object extensions were added
(from github.com/ocaml/ocaml  7974a9d8b1c8195130b2df216d6a102ca00cdf5b^).
It seemed like a good candidate for a fork. I think it was better 
than caml light because caml _special_ light had: 
 - a module system (the possibility to qualify, and #open -> open)
 - a better code organization with code splitted in many directories (parsing/,
   typing/, bytecomp/, etc)
 - it was compatible with lots of my code
It was also far simpler than recent ocaml because it didn't have:
 - an object system! simpler typechecker!
 - labels
 - open variants, and other features that complexify again the typechecker

The size of caml special light 1.16 was also roughly similar to caml light
0.75 when you don't consider asmcomp/ and asmrun/ which again made
it a good candidate for a fork.

But boot/cslc was incorrect!! When compiling its byterun/
and running cslrun or cslrund (via make cslrund) -t -v ../boot/cslc
then I would get a segfault. I think it's because the boot/cslc
is incorrect! I got a segfault in fill_string because the bytecode
in boot/cslc does the wrong call to fill_string. It does a PUSHACC2
instead of a PUSHACC3 because probably its boot/cslc was
compiled when the signature of fill_string in the runtime was different.

This probably made sense when using as a source github.com/ocaml/ocaml
which does not update its boot/ directory at every commit (you
need to use git log on boot/ocamlc or boot/cslc to be sure
to get a commit which updates its bootstrap). But when
using http://caml.inria.fr/pub/distrib/Oldies/csl-1.15.tar.gz
the same thing happened and its byterun/cslrun could not run
its boot/cslc.

Because boot/cslc was incorrect, I tried then to substitute
it by ocamlc from a recent ocaml distribution, but then
to compile csl with ocamlc requires many modifications:
 - many functions like Array.new need to be renamed to Array.create
 - the format special typing trick for printf had to be backported
   and it didn't work completely actually (even with the 
   camlinternalFormatBasic.ml file)
 - at linking time you will get many errors because stdlib/ has
   some external declarations with old C function names (e.g.
   seek_in instead of caml_seek_in), which forced to modify
   most external declarations
 - ...

Enter ocaml 1.07, a version with a working byterun/ocamlrun boot/ocamlc!
finally.

** second try: ocaml 1.07, Dec 1997?

In addition to the fact that ocaml 1.07 is the oldest version
that I could get my hand on with a working byterun/ocamlrun boot/ocamlc,
which makes it obviously the only possible starting point 
candidate for a fork, I think that even though it has the object system
it is maybe also a better candidate in the end for a fork because it has:
 - a byterun/ with ANSI C declarations (and no more the old style prototype)
   => can use pfff -lpize
   (commit 1517cea772058b0fcbe778d05b8b99c7e6f3b25f in github.com/ocaml/ocaml)
 - a stdlib/ with the .create functions instead of the .new 
   => can use pfff -lpize
 - the debugger
 - an emacs mode
 - a better license I think (but still QPL patch-based style)
 - it is ocaml, which makes it simpler to try to compile
   my own project with it (e.g. syncweb). Same commands!

* changes   

** first "cleanup" modification

After opting for ocaml 1.07 aas the starting point, 
I did many modifications, mostly removing stuff:
 - removed support for Windows and Mac (the Makefile.nt, Makefile.Mac, etc)
 - removed native code compiler targets: alpha, hppa, m68k, power, and sparc,
   mips (just kept i386) and I backported the ARM port from ocaml 2.01
 - removed support for objects! reduce the code and the complexity
   of the typechecker. In the end they are not used in the compiler
   (they are very ligtly used in asmcomp/ but the code can be rewritten using
   plain records)
 - removed support for functor! I prefer defunctorized data structures
   anyway. I also removed support for 'include', 'with type' signature, and
   also 'module type' (I never understood them).
   In the end they are also not that used in the compiler, or when
   they are used it's to use the default compare.
 - removed support for streams
 - I reverted to the simpler typechecker (no Link, Nil, expand_head, resetdef,
   newgenty, etc), from caml special light 1.16
   introduced for the need of objects
 - I moved the debugger/ and toplevel/ under tools/
 - I removed some support for macintosh in byterun/ and otherlibs/str
 - I removed some support for win32 in byterun/ and otherlibs/unix
 - I removed the code and the ifdefs for many HAS_XXX
   (HAS_UI, HAS_UNISTD, HAS_MEMMOVE,...) I assume a ansi C and POSIX
   envirionment (and so I assume __STDC__ and _POSIX_SOURCE)
 - I removed some ifdef __alpha

I have also rewritten a few:
 - if then else if --> match () with when
 - iter f data in data |> iter f

** backport stuff from more recent OCaml

Then I backported a few things from a more recent ocaml distribution:
 - config.sub and config.guess and runtest.sh so can have a working
   configure
 - a few modifications to configure to remove HAS_STRERROR test
   and to not use -fno-defer-pop for darwin (clang)
 - b8238a8b61696e454e5fc391dec9d71555b909e9
   PR#5662 : typo in md5.c when clearing sensitive data in context
 - the items listed below

    Objective Caml 3.11.0:
    ----------------------
    
    Standard library:
    - Printexc library: new functions "print_backtrace" and "get_backtrace"
      to obtain a stack backtrace of the most recently raised exception.
      New function "record_backtrace" to turn the exception backtrace mechanism
      on or off from within a program.
      !PARTIAL!

    Objective Caml 3.01:
    --------------------
    Standard library:
    - Module Hashtbl: added Hashtbl.replace.
     !PARTIAL! the DS does not have the size field, so I commented out some code

    Bytecode compiler and bytecode interpreter:
    - Print stack backtrace when a program aborts due to an uncaught exception
      (requires compilation with -g and running with ocamlrun -b or
       OCAMLRUNPARAM="b=1").
    
    Objective Caml 3.00:
    --------------------
    
    Standard library:
    - Module List: added List.sort, List.stable_sort; fixed bug in List.rev_map2.
      !PARTIAL! didn't fix the List.rev_map2 thing
    
    Objective Caml 2.02:
    --------------------
    
    * Standard library:
      - Added the "ignore" primitive function, which just throws away its
        argument and returns "()".  It allows to write
        "ignore(f x); y" if "f x" doesn't have type unit and you don't
        want the warning caused by "f x; y".
        !PARTIAL! wrote a regular function instead of a primitive
      - Added the "Buffer" module (extensible string buffers).
      - Module List: added find, filter, partition.
    
    * The str library:
      - Added split_delim, full_split as variants of split that control
        more precisely what happens to delimiters.
    
    Objective Caml 2.01:
    --------------------
    
    * Standard library:
      - Module Pervasives: added float_of_int (synonymous for float),
        int_of_float (truncate), 
        !PARTIAL! int_of_char (Char.code), char_of_int (Char.chr),  bool_of_string.)

    * C interface:
      - New macros CAMLparam... and CAMLlocal... to simplify the handling
        of local roots in C code.
      - Simplified procedure for allocating and filling Caml blocks from C.
    
    Objective Caml 2.00:
    --------------------
    
    * Language:
      - Record copying with update "{r with lbl1 = expr1; ...}".
        !PARTIAL! opted for a new constructor instead of patching Pexp_record

    * The Unix library:
      - Use float instead of int to represent Unix times (number of seconds
        from the epoch).  This fixes a year 2005 problem on 32-bit platforms.
        Functions affected: stat, lstat, fstat, time, gmtime, localtime,
        mktime, utimes.
      - Added putenv.

    * ocamllex:
      - Added "let <id> = <regexp>" to name regular expressions
        (inspired by Christian Lindig's patch).

    * The str library: added Str.quote, Str.regexp_string,
      Str.regexp_string_case_fold.
